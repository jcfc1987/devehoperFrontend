Documentation

Welcome to the Feigniter documentation. Below you will find information about how to use the library effectively.

Getting Started

Feigniter is a lightweight JavaScript framework designed to simplify the development of dynamic web applications.

Core Features

  • Dynamic routing with hash-based navigation.
  • Controller-based architecture for modular development.
  • Support for caching views, JavaScript, and CSS files.
  • Translation support using i18next.
  • Action registry for handling custom DOM actions.

Configuration

The application configuration is managed through the config.js file. Key settings include:

  • homeController: The default controller to load.
  • defaultMethod: The default method to execute in the controller.
  • useCache: Enable or disable caching for views, JS, and CSS.
  • useTranslation: Enable or disable translation support.

Routing

Feigniter uses hash-based routing. The URL format is:

<code>#ControllerName?methodName=arg1,arg2,...</code>

Example: #HomeController?about

Controllers

Controllers are JavaScript classes that manage the logic for specific pages. Each controller must extend the base Controller class.

Views

Views are HTML files that define the structure of the page. They can be dynamically loaded and cached for better performance.

Translation

Feigniter integrates with i18next for translation support. Translation files are stored in the app/locales directory.

Action Registry

The action registry allows you to define and execute custom actions based on DOM elements with data-feigniter-action-type attributes.

Debugging

Enable debugMode in the configuration to log debug messages and add a "Clear Cache" button for testing purposes.
